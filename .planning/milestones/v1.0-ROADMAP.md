# Milestone v1.0: MVP — Foundation & Gallery Launch

**Status:** SHIPPED 2026-03-28
**Phases:** 1-6
**Total Plans:** 11

## Overview

Built the full pipeline from mathematical computation engine to web gallery: arbitrary-precision math calculations (Fibonacci, primes, fractals) driving headless Blender renders with photorealistic Cycles materials, exported as 4K video, and presented in a Next.js web gallery with academic white paper integration.

## Phases

### Phase 1: Math Engine Foundation (Python Core)

**Goal**: Build the strict mathematical computation engine in Python.
**Requirements:** MATH-01, MATH-02
**Plans**: 1 plan

Plans:
- [x] 01-01: FastAPI server with arbitrary-precision Fibonacci sequence generator

**Details:**
Built FastAPI service with Pydantic models, arbitrary-precision arithmetic via Python's native int, and GET /api/v1/sequences/fibonacci endpoint returning rich JSON with Point3D support.

### Phase 2: Advanced Geometrization & Infinity Computation

**Goal**: Expand the engine to compute spatial vector layouts and point clouds for abstract concepts (Primes, Infinity).
**Requirements:** MATH-03, MATH-04, MATH-05
**Plans**: 2 plans

Plans:
- [x] 02-01: Core geometric math and models (primes, fractals, Point3D)
- [x] 02-02: Geometrization API endpoints (/primes, /fractal)

**Details:**
Implemented 3D Ulam cylinder mapping for primes, L-system fractal evaluation with turtle graphics, and exposed both via new API endpoints with integration tests.

### Phase 3: Render Pipeline Initialization

**Goal**: Set up headless Blender ingestion to reliably read the static mathematical JSON data.
**Requirements:** RNDR-01
**Plans**: 1 plan

Plans:
- [x] 03-01: Pipeline setup — data export script and headless Blender renderer

**Details:**
Created scripts/export_data.py for API data extraction and renderer/render.py for headless Blender object spawning at math-engine coordinates. Installed Blender via Homebrew.

### Phase 4: Aesthetic Surfacing & Materials

**Goal**: Implement the expressive digital art shaders, textures, and materials.
**Requirements:** RNDR-02
**Plans**: 1 plan

Plans:
- [x] 04-01: Photorealistic Cycles rendering with data-driven materials

**Details:**
Applied Cycles renderer with Principled BSDF materials, dynamic coloring by spatial index, HDRI sun lighting, and camera track-to constraint for centroid framing.

### Phase 5: 4K Export Automation

**Goal**: Complete the rendering loop — export image sequences and stitch into broadcast-quality video.
**Requirements:** RNDR-03, RNDR-04
**Plans**: 2 plans

Plans:
- [x] 05-01: Blender 4K frame export (3840x2160, 16-bit PNG, turntable animation)
- [x] 05-02: Pipeline orchestrator and FFmpeg encoding (H.264, CRF 17)

**Details:**
Extended renderer with 4K resolution, 128-sample Cycles with OpenImageDenoise, smooth 360° turntable. Built single-command pipeline orchestrator (scripts/run_pipeline.py) checking dependencies, running Blender headless, and stitching with FFmpeg.

### Phase 6: Web Gallery & White Paper Integration

**Goal**: Build the presentation frontend for 4K videos alongside methodology formatting.
**Requirements:** WEB-01, WEB-02, WEB-03, WEB-04
**Plans**: 4 plans

Plans:
- [x] 06-01: Next.js 16 scaffold, Tailwind v4, Playwright test stubs, data model
- [x] 06-02: VideoPlayer client component with poster-first optimized playback
- [x] 06-03: PaperColumn server component with Pandoc LaTeX-to-HTML pipeline
- [x] 06-04: Gallery assembly — SiteHeader, ArtworkSection, MathJax 4 integration

**Details:**
Shipped static-export Next.js app with responsive gallery layout, optimized video loading (preload=none, poster images), academic paper rendering via Pandoc, and MathJax 4 for formula display.

## Milestone Summary

**Key Decisions:**
- Offline Render + Web Gallery (4K too demanding for real-time web)
- Blender via Homebrew for headless rendering
- Index-based dynamic coloring instead of complex shader math
- Track-to constraint for automatic camera framing
- MathJax 4 CDN for formula rendering
- Static export (output: 'export') for maximum performance

**Issues Resolved:**
- Blender 5.1 compositor API breaks
- 7 compounding render pipeline bugs
- Camera framing issues
- Typography violations in UI spec
