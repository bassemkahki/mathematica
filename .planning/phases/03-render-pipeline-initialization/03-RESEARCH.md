# Phase 3 Research: Render Pipeline Initialization

## Context
Phase 3 requires setting up Blender (headless) to ingest pre-exported JSON files generated by the math engine (`engine/server.py`) and spawn default geometric objects (icospheres) exactly at the 3D coordinates provided.
The coordinates represent points like primes and fractal endpoints. Shaders, materials, animation, and export are out of scope for this phase.

## Technical Approaches

### 1. The Math Engine Endpoint Data Contract
The math engine (`engine/server.py`) exposes `/api/v1/sequences/primes` and `/api/v1/sequences/fractal` returning data corresponding to the Pydantic models in `engine/models.py`.
The rendering pipeline only needs the 3D coordinates which are stored in the output JSON at `data.points` as an array of `{"x": float, "y": float, "z": float}` objects.

We will build an intermediate utility script (`scripts/export_data.py`) to hit these server endpoints and dump the payload into `data/primes.json` and `data/fractal.json`.

### 2. Headless Blender Invocation
Blender should be invoked from the CLI natively without loading a window UI (headless mode):
```bash
blender --background --python renderer/render.py -- --input data/primes.json
```
The arguments after `--` are passed specifically into `sys.argv` inside the python script being executed.

### 3. Mesh Instancing
To render the specific points, we need to iterate over `data.points` in the JSON and instance icospheres.
Using `bpy.ops.mesh.primitive_ico_sphere_add` repeatedly in a loop is very slow for large point clouds because Blender evaluates the dependency graph after every operator call.
**Better approach**: Use instances or create the mesh data directly via `bpy.data.objects.new()` and link it to the scene collection. Since we are just initializing the renderer in this phase, standard object duplication or low-level instancing of a base icosphere is preferred.

## Validation Architecture

To validate Phase 3, we must guarantee:
1. Data ingestion matches the JSON format from Phase 2.
2. Blender creates exactly the expected number of generic objects (matching the length of `points` array).
3. The renderer runs headless without crashing to shell.
