# Plan 01: Photorealistic Shading, Lighting, and Framing

<objective>
Transform the raw geometry generator into a photorealistic rendering scene by implementing data-driven materials, studio lighting, and dynamic camera framing in Blender.
</objective>

<must_haves>
- Script MUST create a node-based Principled BSDF material and apply it to all generated objects.
- Script MUST incorporate data-driven colors for the geometry (e.g. mapping coordinates or indices to `obj.color`).
- Script MUST instantiate at least one Light object (e.g. area or sun light) to illuminate the scene.
- Script MUST add a Camera, set it as the active scene camera, and position it to frame the generated geometry.
- Script MUST set the render engine to CYCLES.
</must_haves>

<step>

<action>
Modify `renderer/render.py` to:
1. Parse an optional `--camera` argument (defaulting to `static`).
2. Set the render engine to CYCLES: `bpy.context.scene.render.engine = 'CYCLES'`.
3. Create a material named `DataMaterial` with `use_nodes=True`. Create a `Principled BSDF` node. Link an `Object Info` node's `Color` output to the Principled BSDF's `Base Color` and `Emission Color` (if desired).
4. Iterate over the points: when calling `primitive_ico_sphere_add`, compute a color `(R, G, B, 1.0)` based on spatial location (e.g., normalized coordinates), set the sphere's `color` attribute to it, and append the `DataMaterial` to the object's `data.materials`.
5. Compute the bounding box of the points (min/max X, Y, Z). Find the centroid.
6. Create an Area or Sun light pointing at the centroid: `bpy.ops.object.light_add(type='SUN', location=(centroid_x + 5, centroid_y - 5, centroid_z + 5))`.
7. Create a camera. Position it along the Y-axis backwards from the centroid by an offset relative to the bounding box size (e.g. distance = max_dimension * 2). Set the camera to point at the centroid.
8. Set `bpy.context.scene.camera = bpy.context.object`.
</action>
