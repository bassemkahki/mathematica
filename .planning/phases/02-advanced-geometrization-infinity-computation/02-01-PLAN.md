### 2. Implement Prime Ulam Cylinder Math
<action>
Create `engine/math_core/primes.py`.
- Write `generate_primes(n: int) -> list[int]` using Sieve of Eratosthenes to return exactly `n` primes.
- Write `generate_ulam_cylinder(n: int, radius: float = 10.0) -> list[dict]` that calculates primes, maps them to `(u, v)` (where u is the prime value, v is the index integer), and projects to 3D cylinder coordinates `x = radius * math.cos(u/radius)`, `z = radius * math.sin(u/radius)`, `y = v`.
- Return the list of dictionaries `[{"x": x, "y": float(y), "z": z}, ...]`
Create `tests/test_primes.py`.
- Write a unit test `test_generate_primes` that asserts `len(generate_primes(5))` is 5 and it equals `[2, 3, 5, 7, 11]`.
- Write `test_generate_ulam_cylinder` asserting the output is a list of dicts with `x`, `y`, and `z` keys.
</action>

### 3. Implement L-system Fractal Engine
<action>
Create `engine/math_core/fractals.py`.
- Write `generate_lsystem(iterations: int, axiom: str="F", rules: dict={"F": "F+F-F-F+F"}) -> str` which expands the L-system string recursively.
- Write `evaluate_lsystem_3d(string_instructions: str, step_length: float=1.0, angle: float=90.0) -> list[dict]` which treats `F` as move forward, `+` as positive yaw rotation, `-` as negative yaw rotation, mapping a 3D turtle graphic state. Returns `[{"x": x, "y": y, "z": z}, ...]`. Pitch and roll are unmodified for 2.5D planar expansion.
Create `tests/test_fractals.py`.
- Write tests that verify string length expansion, and that 3D coordinates are bounded properly from evaluation (it returns points of length proportional to string length).
</action>
