# Plan 02-02: Geometrization API Endpoints

## Objective

## Requirements

## Tasks

### 1. Add Primes API Endpoint
<action>
Modify `engine/server.py`.
- Import `generate_ulam_cylinder` from `engine.math_core.primes`.
- Add a new `GET` route `@app.get("/api/v1/sequences/primes", response_model=SequenceResponse)`.
- The endpoint takes query param `n: int`.
- It executes `points = generate_ulam_cylinder(n)`, calculates time, and returns a `SequenceResponse` with `metadata` containing `sequence_type="primes", generation_bounds={"max_n": n}, calculation_time_ms=...`, and `data` constructed using `points=points, sequence=[]` (or similar depending on optionality).
Modify `tests/test_server.py` (create if doesn't exist).
- Add tests using `fastapi.testclient.TestClient` verifying `GET /api/v1/sequences/primes?n=5` returns HTTP 200, checks the response JSON contains `x, y, z` under `data.points[0]`.
</action>

### 2. Add Fractal API Endpoint
<action>
Modify `engine/server.py`.
- Import `generate_lsystem` and `evaluate_lsystem_3d` from `engine.math_core.fractals`.
- Add a new `GET` route `@app.get("/api/v1/sequences/fractal", response_model=SequenceResponse)`.
- The endpoint takes query param `iterations: int`.
- Computes `lsys = generate_lsystem(iterations)` and `points = evaluate_lsystem_3d(lsys)`.
- Returns `SequenceResponse` with `metadata.sequence_type="fractal", metadata.generation_bounds={"iterations": iterations}`, and `data.points=points`.
Modify `tests/test_server.py`.
- Add a test checking `GET /api/v1/sequences/fractal?iterations=2` verifying status code 200 and points structure length greater than 0.
</action>
