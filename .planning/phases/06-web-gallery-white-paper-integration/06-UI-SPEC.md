# Phase 6 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — hand-authored with Tailwind CSS |
| Icon library | none — single SVG play icon inline |
| Font | Inter (Google Fonts CDN) — system-ui fallback |

**Rationale (Claude's discretion):** This is a single-scroll, cinematic gallery with no interactive component complexity (no modals, dropdowns, data tables, or forms). shadcn adds overhead without payoff for this surface. Tailwind CSS gives full layout control over the full-width video/paper side-by-side sections with minimal bundle cost. MathJax CDN is already mandated by D-03 for equations; adding a second CDN (Google Fonts) is acceptable.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding within labels |
| sm | 8px | Compact element spacing (poster caption, label to border) |
| md | 16px | Default element spacing (paragraph gap, control padding) |
| lg | 24px | Section padding within an artwork block |
| xl | 32px | Gap between video column and paper column |
| 2xl | 48px | Padding above/below each full-width artwork section |
| 3xl | 64px | Separation between successive artwork sections |

Exceptions: Video touch target (play overlay button) — minimum 44×44px to meet WCAG 2.1 touch target requirement.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body (paper prose) | 16px | 400 (regular) | 1.6 |
| Label (artwork title, section heading) | 20px | 600 (semibold) | 1.3 |
| Heading (page-level, artwork series name) | 28px | 600 (semibold) | 1.2 |
| Display (site title / hero) | 40px | 600 (semibold) | 1.1 |

**Source:** Default — academic editorial aesthetic requires generous body line-height (1.6 vs standard 1.5) for comfortable paper reading alongside dense math notation.

MathJax-rendered equations inherit Inter font stack and render at 1em relative to their containing body text.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #0A0A0A | Page background, video section background |
| Secondary (30%) | #141414 | White paper column background, artwork meta areas |
| Accent (10%) | #E8E8E8 | Play button icon, artwork titles, active state indicator on play control |
| Destructive | not applicable | No destructive actions in this phase |

Accent reserved for: play button SVG icon, artwork title text, the poster-to-video transition control border-on-hover only. Body prose text uses #C8C8C8 (near-white, not full accent) to preserve contrast hierarchy.

**Color rationale:** Dark near-black dominant is required by the cinematic/mathematical art aesthetic described in CONTEXT.md (`## Specific Ideas`: "standard approaches consistent with the academic/mathematical art aesthetic"). High contrast off-white accent on near-black background exceeds WCAG AA 7:1 ratio. Body prose is slightly dimmed (#C8C8C8) to reduce eye strain on long paper reads.

## Layout Contract

*This section supplements color/type with the structural decisions from D-04.*

### Desktop (≥ 1024px)
- Full-width artwork section: `display: grid; grid-template-columns: 1fr 1fr; gap: 32px`
- Video column left, white paper column right
- Section max-width: 1440px, centered with `margin: 0 auto; padding: 0 48px`

### Tablet (768px – 1023px)
- Grid collapses to single column
- Video block appears above paper block
- Section padding reduces to `0 24px`

### Mobile (< 768px)
- Single column, full-width
- Section padding reduces to `0 16px`
- Video aspect ratio maintained via `aspect-ratio: 16/9` on wrapper

**Source:** D-04 (single-scroll, side-by-side on desktop, video stacks above paper on mobile) + Claude's discretion on exact breakpoints.

## Video Interaction Contract

*Poster-first, user-triggered playback — mandated by D-02.*

### Default State (Poster)
- `<video>` element renders with `poster` attribute pointing to PNG still from `renders/test_frames/`
- `preload="none"` — no buffering until user plays
- Overlay: centered play button (44×44px minimum, circular, `background: rgba(255,255,255,0.15)`, white SVG triangle icon at 20px)
- Overlay appears at 0.7 opacity at rest, 1.0 opacity on hover (`transition: opacity 150ms ease`)

### Triggered State (Playing)
- On play-button click: overlay fades out (`opacity 0, pointer-events none`) over 200ms
- `video.play()` called programmatically
- Native browser video controls appear (`controls` attribute toggled on) after overlay fade completes
- Controls styled via Tailwind — no custom CSS for native controls (cross-browser constraint)

### Error State
- If video fails to load: poster image remains visible
- Below video: "Video unavailable — file may still be rendering." in 14px / #888 italic
- No retry mechanism in v1

## White Paper Column Contract

*LaTeX → Pandoc → HTML fragment — mandated by D-03.*

- Pandoc-produced HTML fragments are embedded inside Next.js page as `dangerouslySetInnerHTML` (static build-time only, no user input, safe for static HTML)
- MathJax loaded from CDN: `https://cdn.jsdelivr.net/npm/mathjax@3/es5/tex-chtml.js` with `defer`
- Paper column: `overflow-y: auto; max-height: 90vh` on desktop — enables independent scroll of long papers beside fixed-aspect video
- Typography: body at 16px / 400 / line-height 1.6 / color #C8C8C8 on #141414 background
- `h1`, `h2`, `h3` within paper fragments use artwork title scale (20px / 600) — Pandoc heading levels mapped via Tailwind prose overrides
- `blockquote` and `pre/code` blocks: `background: #1E1E1E; border-left: 3px solid #E8E8E8; padding: 8px 16px`
